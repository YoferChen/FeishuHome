# 飞书多维表格设计文档

本文档详细说明了用于管理飞书博客主页的多维表格结构设计。

## 概述

为了实现首页配置的动态管理，我们将网站配置从硬编码的配置文件迁移到飞书多维表格。这样可以：

- 无需修改代码即可更新网站内容
- 通过飞书界面方便地管理所有配置
- 支持多人协作编辑
- 实现配置的版本控制和审批流程

## 表格结构设计

### 1. 网站基础配置表 (Site Config)

管理网站的基本信息和全局配置。

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| 配置项 | 单行文本 | ✓ | 配置项的唯一标识 | `site_title` |
| 配置值 | 多行文本 | ✓ | 配置项的值 | `我的个人博客` |
| 配置类型 | 单选 | ✓ | 值的数据类型 | `文本`/`数字`/`布尔值`/`URL` |
| 分组 | 单选 | ✓ | 配置所属分组 | `基本信息`/`SEO`/`飞书配置` |
| 描述 | 多行文本 | - | 配置项的说明 | `网站的标题，显示在浏览器标签页` |
| 是否启用 | 复选框 | ✓ | 是否启用该配置 | `true` |
| 更新时间 | 日期 | - | 最后更新时间 | `2026-02-27` |

**预设配置项：**

```
基本信息分组：
- site_title: 网站标题
- site_description: 网站描述
- site_keywords: SEO 关键词
- site_author: 作者名称
- site_url: 网站 URL

飞书配置分组：
- feishu_app_id: 飞书应用 ID
- feishu_app_secret: 飞书应用密钥
- feishu_table_id_blogs: 博客表格 ID
- feishu_table_id_config: 配置表格 ID
```

## 数据获取流程

### 1. 环境变量配置

在 `.env` 文件中添加所有表格的 ID：

```env
# 飞书应用配置
FEISHU_APP_ID=your_app_id
FEISHU_APP_SECRET=your_app_secret

# 多维表格 ID
FEISHU_TABLE_ID_BLOGS=博客表格ID
FEISHU_TABLE_ID_SITE_CONFIG=网站配置表格ID
FEISHU_TABLE_ID_PROFILE=个人信息表格ID
FEISHU_TABLE_ID_EDUCATION=教育经历表格ID
FEISHU_TABLE_ID_PROJECTS=项目经历表格ID
FEISHU_TABLE_ID_MILESTONES=里程碑表格ID
FEISHU_TABLE_ID_THEME=主题样式表格ID
```

### 2. 数据获取脚本

创建 `scripts/fetch-config.ts` 脚本，用于从飞书多维表格获取所有配置数据。

### 3. 生成的数据文件

脚本将生成以下 JSON 文件：

- `src/data/site-config.json` - 网站基础配置
- `src/data/profile.json` - 个人信息
- `src/data/education.json` - 教育经历列表
- `src/data/projects.json` - 项目经历列表
- `src/data/milestones.json` - 里程碑列表
- `src/data/theme.json` - 主题样式配置

### 4. 前端集成

前端组件将从这些 JSON 文件读取数据，而不是从硬编码的配置文件。

## 实施步骤

1. **在飞书中创建表格**
   - 按照上述结构创建 6 个多维表格
   - 填入初始数据

2. **配置环境变量**
   - 在 `.env` 中添加所有表格 ID
   - 在 GitHub Secrets 中配置（用于 CI/CD）

3. **开发数据获取脚本**
   - 创建 `scripts/fetch-config.ts`
   - 实现从飞书 API 获取数据的逻辑
   - 处理数据转换和验证

4. **更新前端代码**
   - 修改组件以使用新的数据源
   - 实现主题样式的动态应用
   - 添加数据加载和错误处理

5. **更新 CI/CD 流程**
   - 在 GitHub Actions 中添加配置数据获取步骤
   - 确保构建时自动更新所有数据

6. **测试验证**
   - 测试数据获取脚本
   - 验证前端显示效果
   - 测试主题样式切换

## 优势

1. **内容管理便捷**：通过飞书界面即可更新所有内容，无需修改代码
2. **协作友好**：支持多人同时编辑，有权限控制
3. **版本控制**：飞书自带版本历史，可追溯修改记录
4. **审批流程**：可配置审批流程，确保内容质量
5. **动态主题**：支持动态切换主题样式，无需重新部署
6. **数据分离**：配置数据与代码分离，降低维护成本

## 注意事项

1. **数据验证**：获取数据后需要进行格式验证，确保数据完整性
2. **错误处理**：需要处理 API 调用失败、数据格式错误等异常情况
3. **缓存策略**：考虑添加数据缓存，减少 API 调用次数
4. **安全性**：敏感配置（如 API 密钥）不应存储在表格中
5. **性能优化**：大量数据时考虑分页加载或懒加载
6. **向后兼容**：保留原有配置文件作为降级方案
